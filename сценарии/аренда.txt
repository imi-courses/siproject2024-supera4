# - ошибки
@ - альтернативный сценарий

Просмотреть аренды
1.  Пользователь находится в главном окне
2.  Пользователь выбирает раздел список аренд
3.  Пользователь находится в окне списка аренд
#Если СУБД не открывается, то выводится окно с сообщением об ошибке

Добавить аренду
1.  Пользователь находится в окне списка аренд
2.  
    - Пользователь нажимает на кнопку добавить
    - Открывается форма ввода данных
    - Пользователь вводит данные аренды
    - Пользователь нажимает сохранить
    - Программа выводит окно подтверждения
    - Пользователь нажимает на кнопку подтвердить
    - Программа добавляет аренду в базу данных
    - Форма ввода данных закрывается
3.  Пользователь находится в обновленном окне списка аренд
#Если пользователь вводит неправильные символы, то выводится сообщение с разрешёнными символами в поле
#Если пользователь не заполняет все обязательные поля, то выводится окно с сообщением "не все обязательные поля заполнены"
#Если аренда с такими данными в БД уже существует, то выводится окно с сообщением "такая аренда уже существует"
#Если СУБД не открывается, то выводится окно с сообщением об ошибке
@Если пользователь нажимает кнопку отмена, то окно подтверждения закрывается

Удалить аренду
1.  Пользователь находится в окне списка аренд
2.  
    - Пользователь выбирает аренду(ы)
    - Пользователь нажимает на кнопку удалить
    - Программа выводит окно подтверждения
    - Пользователь нажимает на кнопку подтвердить
    - Программа удаляет аренду(ы) из базы данных
3.  Пользователь находится в обновленном окне списка аренд
#Если пользователь случайно удаляет не то что нужно, то можно использовать сочетание клавиш ctrl+z.
#Если пользователь нажимает на кнопку отменить, то возвращается в окно списка аренд с выбранными арендами
#Если СУБД не открывается, то выводится окно с сообщением об ошибке
@Если пользователь нажимает кнопку отмена, то окно подтверждения закрывается

Поиск аренды
1.  Пользователь находится в окне списка аренд
2.  Пользователь вводит в поле для поиска данные аренды
3.  Пользователь находится в окне найденных аренд
#Если программа ничего не находит, то выводится сообщение "по вашему запросу ничего не найдено"
#Если пользователь вводит неправильные символы, то выводится сообщение с разрешёнными символами в поле
#Если СУБД не открывается, то выводится окно с сообщением об ошибке

Редактировать аренду
1.  Пользователь находится в окне списка аренд
2.  
    - Пользователь выбирает аренду
    - Пользователь нажимает на кнопку редактировать
    - Открывается форма ввода данных, уже заполненная данными выбранной аренды
    - Пользователь редактирует нужные данные
    - Пользователь нажимает на кнопку сохранить
    - Программа выводит окно подтверждения
    - Пользователь нажимает на кнопку подтвердить
    - Программа сохраняет новые данные в базу данных
    - Форма ввода данных закрывается
3.  Пользователь находится в обновленном списке аренд
#Если пользователь выбирает несколько аренд, то кнопка редактировать будет неактивна (тускнеет)
#Если пользователь вводит неправильные символы, то выводится сообщение с разрешёнными символами в поле
#Если пользователь не заполняет все обязательные поля, то выводится окно с сообщением "не все обязательные поля заполнены"
#Если аренда с такими данными в БД уже существует, то выводится окно с сообщением "такая аренда уже существует"
#Если СУБД не открывается, то выводится окно с сообщением об ошибке
@Если пользователь нажимает кнопку отмена, то окно подтверждения закрывается

Продлить аренду
1. Пользователь находится в окне списка аренд
2. 
    - Пользователь выбрирает аренду
    - Пользователь нажимает на кнопку продлить
    - Программа открывает окно со сроком продления
    - Пользователь вводит срок продления
    - Пользователь нажимает продлить
    - Программа выводит окно подтверждения
    - Пользователь нажимает на кнопку подтвердить
    - Программа продливает аренду
3. Пользователь находится в обновленном окне списка аренд
#Если пользователь выбирает несколько аренд, то кнопка продлить будет неактивна (тускнеет)
#Если СУБД не открывается, то выводится окно с сообщением об ошибке
@Если пользователь нажимает кнопку отмена, то окно подтверждения закрывается


Закрыть аренду
1. Пользователь находится в окне списка аренд
2. 
    - Пользователь выбрирает аренду
    - Пользователь нажимает на кнопку закрыть аренду
    - Программа выводит окно подтверждения
    - Пользователь нажимает на кнопку подтвердить
    - Программа закрывает аренду
3. Пользователь находится в обновленном окне списка аренд
#Если пользователь выбирает несколько аренд, то кнопка закрыть аренду будет неактивна (тускнеет)
#Если СУБД не открывается, то выводится окно с сообщением об ошибке
@Если пользователь нажимает кнопку отмена, то окно подтверждения закрывается
